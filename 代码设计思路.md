# **代码设计思路 **
## **整体要求**
这个项目只是一次软件开发技术基础课程实验练习，要求采用面向对象思想进行架构设计，合理定义类（如 DataReader、DataAnalyzer、ResultExporter 等），封装数据属性与方法，体现高内聚、低耦合的设计原则。
1.使用Python实现，代码要清晰、容易看懂，代码的实现方式要简洁，更多集中在算法的逻辑实现和展示上，而不是考虑过于复杂的python库和依赖。
2.在数据读取与存储环节，需根据数据特征选择适宜的数据结构（如列表、字典或自定义对象），确保数据操作的效率。
3.完成系统相关核心功能，算法逻辑需自行实现或合理封装，体现对算法原理的理解，以及在现有方法之上的改进和创新。
4.每个模块都要有简单注释，方便我后续review和作业提交时供批阅老师阅读。
5.将系统功能运行结果以清晰、规范的形式展示、文件保存。

## **实验内容：基于机器学习的混凝土抗压强度预测系统**

### 系统需求分析

1.1 背景与目标
在建筑工程设计与施工中，混凝土抗压强度是衡量结构安全的关键指标。传统方法依赖于实验室试块养护和破坏性试验，周期长、成本高。利用人工智能技术，基于混凝土配合比（水泥、高炉矿渣、粉煤灰、水、减水剂、粗骨料、细骨料）和龄期等特征，预测其抗压强度，可辅助工程人员快速评估混凝土质量，优化配比设计，具有重要的实用价值。
本实验旨在开发一个完整的 混凝土抗压强度预测系统，采用面向对象思想进行架构设计，集成数据读取、预处理、模型训练与评估、结果可视化与导出等模块。系统需支持从CSV文件读取数据，提供多种回归算法（如线性回归、决策树回归），并能评估预测性能。最终通过图形界面（GUI）展示预测结果与分析图表，并将结果保存为文件。

1.2 功能需求
数据输入：支持读取混凝土抗压强度数据集（CSV格式，包含8个特征列和1个目标列）。对数据进行基本的统计描述。
数据预处理：可选择标准化特征、划分训练集与测试集。
模型训练与评估：
   提供至少两种算法：多元线性回归、决策树回归。
   用户可调整模型参数（如决策树深度）。
   训练后计算并显示均方误差（MSE）、决定系数（R²）等指标。
结果展示：通过 matplotlib 图表展示真实值 vs 预测值散点图、残差分布图；在界面中显示评估指标数值。
结果导出：将模型评估报告和预测结果保存为TXT或CSV文件。
交互界面：提供基于 tkinter 的图形界面，整合上述功能，操作简便。
异常处理：对文件不存在、数据格式错误、模型训练失败等情况进行捕获并友好提示。

1.3 非功能需求
面向对象设计：合理抽象类，封装数据与行为，降低耦合。
代码规范：符合PEP8标准，注释清晰。
可测试性：核心算法独立封装，便于编写单元测试。
版本管理：使用Git进行版本控制，提交信息明确。

### 系统总体架构设计
2.1 系统架构
系统采用分层架构，分为表示层、业务逻辑层和数据访问层，通过面向对象的方式组织。
表示层（UI）：负责用户交互，调用业务逻辑，展示结果。由 MainApplication 类（继承自 tk.Tk）管理窗口和控件。
业务逻辑层：包含数据预处理、模型训练、评估等核心功能。由 DataAnalyzer、ModelTrainer、Evaluator 等类协作完成。
数据访问层：负责从文件读取数据和将结果写入文件。由 DataReader 和 ResultExporter 类实现。

2.2 类设计（面向对象）
类名	职责	主要属性	主要方法
DataReader	读取CSV数据集，返回特征和目标数组	file_path	load_data() → (X, y, feature_names)
DataPreprocessor	数据清洗、标准化、划分训练/测试集	scaler (可选)	standardize(X), split(X, y, ratio)
ModelTrainer	训练指定的回归模型	model, model_name, parameters	train(X_train, y_train), predict(X_test) → y_pred
Evaluator	计算评估指标（MSE, R2）	无	evaluate(y_true, y_pred) → dict
ResultExporter	将评估报告、预测值保存到文件	无	export_report(metrics, path), export_predictions(y_true, y_pred, path)
MainApplication	GUI主窗口，整合各模块，响应事件	reader, analyzer, trainer, evaluator 等	__init__, load_file(), train_model(), show_results()

2.3 数据结构选择
数据集内部存储采用 NumPy 数组 或 Pandas DataFrame，以利用其高效的数值运算和切片功能。考虑到轻量化，可选用纯列表+手动运算，但推荐使用 NumPy（科学计算基础库）。
模型参数用 字典 存储，如 {'model_type': 'decision_tree', 'max_depth': 5}，方便扩展。
评估指标结果用 字典 返回，如 {'MSE': 125.3, 'R2': 0.82}。
特征名称用元组或列表保持顺序。

2.4 核心算法设计
2.4.1 多元线性回归（自行实现梯度下降）
为体现算法理解，可自行实现基于梯度下降的线性回归，不使用 sklearn。
损失函数：均方误差（MSE） J = (1/2m) Σ(h(x) - y)^2。
梯度下降更新：θ = θ - (α/m) X^T (Xθ - y)。
添加正则化选项（L2）防止过拟合。

2.4.2 决策树回归（自行实现CART）
基于二叉递归分裂。选择分裂特征和分裂点，使得分裂后的两个子集的MSE之和最小。
停止条件：达到最大深度、节点样本数小于阈值或MSE降低小于阈值。
预测时，遍历树到叶子节点，返回叶子节点目标均值。
（说明：为了代码简洁和可靠，也可以使用 sklearn 作为后备，但实验要求体现算法逻辑，因此核心算法自行实现，并在报告中详细描述推导过程。）

2.4.3 数据标准化（Z-Score）
对每个特征列计算均值 μ 和标准差 σ，转换 x' = (x - μ) / σ。

2.5 界面设计
GUI 使用 tkinter，布局如下：
顶部：文件选择区域（“打开数据文件”按钮，显示文件路径）。
左侧参数设置区：选择算法（下拉框），配置参数（如决策树深度输入框），训练/测试分割比例滑动条。
右侧结果展示区：使用 matplotlib 嵌入 FigureCanvasTkAgg 显示预测 vs 真实值散点图；下方显示评估指标文本。
底部：操作按钮（“开始训练”、“导出结果”、“清除”），状态栏显示当前状态。
